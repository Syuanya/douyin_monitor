# 内容监控批量导入与批量处理优化

本轮优化聚焦内容监控页的批量导入、检测/同步反馈和风控提示。

## 批量导入

- 批量导入入口支持文本粘贴和 TXT / CSV 文件导入。
- 导入前会先生成预览，展示新增、更新、重复、无效行数量。
- 重复检测按规范化主页 URL 和 `sec_user_id` 判断，避免同一账号因跟踪参数或末尾斜杠被重复导入。
- 默认不再“导入后立即开始监控”，避免大批量导入后立刻触发大量请求。
- 导入结果会写入最近批量操作明细，失败行会保留原因。

## 批量处理

- 顶部“批量选择”改为“批量处理”，语义更贴近批量检测、同步、设置、开始/停止监控和删除。
- 批量模式文案改为“批量处理：当前列表已选...”。

## 检测 / 同步 / 刷新

- “刷新”仅刷新本地界面，不请求抖音；点击后会明确提示。
- “检测更新”与“同步作品”的 tooltip 说明请求成本差异。
- “同步全部作品”增加二次确认，降低误触和风控风险。
- 批量检测 / 同步底层接入取消检查，取消后会尽快停止未开始的账号。

## 失败反馈

失败明细会追加下一步建议：

- 疑似 Cookie / 登录态异常：提示更新 Cookie。
- 疑似 IP / 风控限制：提示暂停批次、降低并发并等待冷却。
- 主页不可访问或无公开作品：提示打开主页确认。
- 用户取消：标注为取消状态。
